# Ansible Playbooks - Reference Archive

This directory contains Ansible playbooks that were created as an alternative deployment method.

## Status: ARCHIVED

These playbooks are **not used** by the main deployment script (`deploy-k8s-cluster.ps1`).

## Why Archived?

- Ansible has compatibility issues on Windows PowerShell
- The main script uses direct SSH commands instead
- Kept for reference and alternative deployment scenarios

## Contents

- `ansible.cfg` - Ansible configuration
- `inventory.ini` - Generated by Terraform (dynamic)
- `site.yml` - Main playbook orchestrating all tasks
- `tasks/` - Individual task files:
  - `common.yml` - System updates, Docker installation
  - `k3s-master.yml` - Master node k3s setup
  - `k3s-worker.yml` - Worker node k3s agent setup
  - `argocd.yml` - Argo CD installation

## If You Want to Use These

**Prerequisites:**
- Linux or macOS (or WSL2 on Windows)
- Ansible installed: `pip install ansible`
- Terraform outputs generated

**Run:**
```bash
# Full deployment
ansible-playbook site.yml

# Specific components
ansible-playbook site.yml --tags common
ansible-playbook site.yml --tags master
ansible-playbook site.yml --tags worker
ansible-playbook site.yml --tags argocd
```

## Recommended Approach

Use the main PowerShell script instead:
```powershell
.\deploy-k8s-cluster.ps1
```

This is simpler, works on Windows, and handles everything end-to-end.
